# Getting Started

Before contributing to the SDK, read the main [`CONTRIBUTING.md`](https://github.com/rhesis-ai/rhesis/blob/main/CONTRIBUTING.md) at the root of the repository. It contains guidelines that apply to all contributors, including commit conventions and the pull request workflow.

## Before you start

Install [`uv`](https://docs.astral.sh/uv/) for Python environment management.

## Development setup

<CodeBlock filename="setup.sh" language="bash">
{`git clone https://github.com/rhesis-ai/rhesis.git
cd rhesis/sdk

# Create a development virtual environment
uv sync --dev

# Activate the virtual environment
source .venv/bin/activate`}
</CodeBlock>

## Linting and formatting

From `sdk`:

<CodeBlock filename="format-and-lint.sh" language="bash">
{`make format      # auto-format and fix all Python files
make lint        # check formatting without modifying files`}
</CodeBlock>

Both targets use [ruff](https://docs.astral.sh/ruff/) via `uvx`. There are also diff-scoped variants that only run against files changed relative to `main`:

<CodeBlock filename="format-and-lint-diff.sh" language="bash">
{`make format_diff
make lint_diff`}
</CodeBlock>

## Tests

Tests live in `../tests/sdk` (relative to `sdk/`), not inside the SDK package itself.

From `sdk`:

- **Unit tests** — `make test` runs `pytest` on `../tests/sdk`, ignoring `../tests/sdk/integration`.
- **Integration tests** — `make test-integration` starts the `sdk` Docker profile (see `tests/docker-compose.test.yml`), then runs the full suite including `../tests/sdk/integration`.
- **Coverage** — `make test-coverage` runs `pytest` with coverage reporting for `src/rhesis`.
- **Teardown** — `make docker-down` (or `make docker-clean` to remove volumes).

Run a single test after starting the Docker profile yourself:

<CodeBlock filename="run-specific-test.sh" language="bash">
{`make docker-up
uv run pytest -s ../tests/sdk/entities/test_endpoint.py::test_endpoint -v`}
</CodeBlock>
