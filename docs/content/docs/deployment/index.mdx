## Choosing a Strategy

- **Rhesis Cloud (Recommended)**: Best for most teams. Zero setup, instant access, and fully managed.
- **Start Locally**: If you are exploring Rhesis for the first time, want to test integrations, or only need to evaluate local applications, [Running Locally](/deployment/quick-start) with Docker Compose is the recommended path. It sets up the database, message broker, backend, and frontend in minutes.
- **Move to Production**: When you are ready to scale, need high availability, or want to make Rhesis securely available across your team on your own hardware, follow the [Self-Hosting](/deployment/docker-compose) guide. It covers advanced configurations like external databases, persistent storage, and production-ready ingress.
