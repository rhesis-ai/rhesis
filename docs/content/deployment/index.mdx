# Deployment

While **we strongly recommend using Rhesis Cloud** for the fastest, most reliable experience without any operational overhead, we understand that some teams have strict data privacy requirements or prefer to manage their own infrastructure.

For anyone who does not want to run Rhesis in the cloud, we provide these robust self-deployment options.

<Callout type="info">
  **Want the easiest path?**
  Skip the infrastructure management and start testing immediately with [Rhesis Cloud](https://app.rhesis.ai). We handle the hosting, scaling, and updates so you can focus on building great AI apps.
</Callout>

## Choosing a Deployment Strategy

- **Rhesis Cloud (Recommended)**: Best for most teams. Zero setup, instant access, and fully managed.
- **Start Locally**: If you are exploring Rhesis for the first time, want to test out integrations, or only need to evaluate local applications, [Running Locally](/deployment/running-locally) with Docker Compose is the recommended path. It sets up the database, message broker, backend, and frontend in minutes.
- **Move to Production**: When you are ready to scale, need high availability, or want to make Rhesis available across your team securely on your own hardware, follow the [Self-Hosting](/deployment/self-hosting) guide. It covers advanced configurations such as setting up external databases, persistent storage, and production-ready ingress.
