# Behaviors

Define what good looks like for your AI system and attach metrics to measure it.

<Callout type="default">
  **What are behaviors?** Behaviors are expectations for how your application should perform — for example, response quality, safety, or brand voice. You can group different [Metrics](/docs/metrics) by behavior.
</Callout>

## Behaviors and metrics

Rhesis uses a two-layer model: **behaviors** state what you expect; **metrics** verify it. One behavior can have multiple metrics so you can measure the same expectation from different angles.


Tests are tagged with a behavior so results roll up by expectation. See [Tests](/docs/tests) and [Test Results](/docs/test-results).
